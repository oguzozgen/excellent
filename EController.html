depends(ctrlNames)

Verifies that each controller in the list of dependencies exists, or else throws an error.

This optional level of verification is useful when sub-controllers are rarely used, or loaded dynamically. Such explicit verification makes the code more robust.

post(data, cbopt)

Asynchronously sends data into this controller:

  • If this controller implements onReceive handler, the returned value will be passed into optional cb - callback function.
  • And if this controller does not implement onReceive handler, the method will do nothing.

This method requires that controller is initialized.

onDestroy

De-initialization event handler.

It signals the controller that its element has been removed from the DOM, and it is time to release any pre-allocated resources, if necessary.

For any modern browser, the event is triggered automatically, courtesy of MutationObserver, while for older browsers, such as IE9 and IE10 it falls back on a manual background check that runs every 500ms.
